HTML入门基础知识？

从知乎上看到的一个关于“若想学HTML，应从何入手？”的回答，虽然答案很劝退，但对于我这种非专业想跨行转计算机的来说，却也是一个补充自己专业基础知识的好机会，故以写此博客为目的查找这些基础知识的概念。
原文链接网页打开需要下拉几下，答案大概排在第5个的样子

WEB基础知识（了解）

一 WEB与Internet

WEB和Internet都被中文翻译成网络的意思，其实，它们各有所指
先说Internet，Internet的意思在这句话中体现的最为明显：“全球性的计算机互联网络，指各个子网所共同连接起来的一个互联的网，重点在于信息共享
而WEB重点指依附于Interne所形成的网页应用等。WEB本意是蜘蛛网和王的意思，表现为三种形式：超文本、超媒体和超文本传输协议
两者之间的关系：
web是Internet上最流行的应用之一，Internet为web提供了网络运行环境
而web的出现极大地推动了Internet的普及和推广

1 Internet

译作因特网，指全球性计算机互联网络，又叫互联网、国际互联网。
网际网络是指在广域网与广域网之间互相连接的网络，互联网的概念也包含在其中，网际网络的概念更抽象一些，指像互联网一样连接起来起来形成的网。

二 Internet提供的服务

主要提供四种服务：
高级浏览WWW服务
电子邮件E-mail服务
远程登录Telent服务
以及文件传输FTP服务

1 高级浏览WEB服务

依附于Internet所形成的的网页应用，主要形式有超文本、超媒体和超文本传输协议

2 电子邮件E-mail服务

字面意思，用邮箱收发邮件的功能，是互联网早期的主要服务之一

3 远程登录Telent

Telent协议是TCP//议族中应用层协议之一，是Interne提供的用来对本地或远端运行的网络设备进行管理、配置、监听和维护，这提高了用户的灵活操作性。模拟管理员进行远程登录好像是以前黑客常用的技术手段，要用到非对称加密技术好像，不太好操作

4 FTP

FTP是与HTTP同层次的在Internet上别广泛使用的协议，HTTP是超文本传输协议，而FTP是文件传输协议，FTP的实现比HTTP的实现要复杂

5 BBS:电子公告板(论坛)

英文Bulletin Board System的缩写，即电子公告板，互联网早期主要应用之一，现在也在被广泛应用着

三 实现技术

1 分组交换原理

就是将用户之间交流的大段的信息拆分成有规定的长度限制的小段，每个小段分为头部和数据段，头部包含有发送方和接收方的地址等信息，而用户传输的数据就在数据段。发送的时候小段在网络中传输不分顺序，在交换机中先缓存起来，等待交换机按照当前的网络状况安排合适的传输线路。等到所有小段都到达目的地后再按顺序整合还愿信息，这就是分组交换原理。

2 TCP/IP 协议簇

先贴一下百度百科中对于这个协议簇的定义：tcp/ip协议簇,可以看到这个协议簇是以TCP/IP协议为主还有其它众多协议一起构建了Internet的基础，包含有SLIP协议、PPP协议、IP协议、ICMP协议、ARP协议、TCP协议、UDP协议、FTP协议、DNS协议、SMTP协议等协议。对这些协议的讲解我将在另外写一篇博客来说明链接。

四 WEB程序

1 什么是WEB

本来想借用上边写的区分WEB和Internet的说明来讲解这个问题，但是看上边写的也是很不具体，那就在这边好好解释一下吧。
WEB主要就是指网页了，在Internet这个将世界各地的局域网搭建成一个更大的、统一的、一个整体的互相连接的一个internet基础上，人们利用HTTP超文本传输协议等一系列协议创建了以文件和超链接将Internet上的各个信息节点连接起来，早期的网页只是由简单的文字和连接，后期虽然图片、音频、视频等表现形式更加直观的文本被展现在网页上，但WEB的基本组成依然还是文本和超链接。

WEB的工作原理

一 基于B/S模式的程序（Web）

1.B/S

B/S就是“Browser/Server”的缩写，B是浏览器的意思，而S是服务器的意思，即浏览器/服务器结构，是对以往的C/S架构的一种改进，也可以说是三层的C/S结构，是一种全新的软件系统构造技术。
第一层：指浏览器，即客户端，只有简单的输入输出功能，本身处理极少的事务逻辑，面向大范围的用户。
第二层：指WEB服务器，帮忙处理第一层客户端与第三层数据库服务器之间的信息调度。
第三层：服务数据库，不仅存储着大量的原始数据，还要处理主要的事务逻辑，是服务器的核心。

2.基于C/S模式的程序

补充一下B/S的前身C/S：
C/S就是“Client/Server”的缩写，C指客户端，S指服务器，所以就是客户端/服务器结构。之前我以为随着B/S的流行，C/S已经落后被淘汰了，其实不是，我们手机中的微信、QQ、王者荣耀等app其实就是C/S结构的。B/S的发展并不是C/S的升级，而是属于从C/S结构中分化出来的针对WEB的高效率的工作的一种分化。

3.二者的区别：

1 硬件环境不同：C/S建立在专门的网络上，有专门的客户端才能访问服务器，而B/S则是建立在广域的网络之上的，任何一款操作系统或浏览器都可以访问服务器。
2 由于上一条两者硬件环境的不同，而导致了本条C/S比B/S结构更安全的不同，因为C/S用户更固定，服务器信息更安全
3 B/S结构维护升级更麻烦一些，动不动就要下载海量的资源包来更新，要完成升级需要客户端和服务器端都升级，而B/S结构升级只需要服务器端完成升级即可。

服务器与浏览器

一 服务器

1.主要功能

给用户提供服务的，根据服务器提供的服务类型不同，分为文件服务器，数据库服务器，应用程序服务器，WEB服务器等

  • 存储WEB信息，提供管理环境
  • 响应浏览器的请求，执行服务器端程序
  • 安全功能

2.主要产品

  1. TOMCAT
    目前比较流行的WEB应用服务器，由APACHE软件基金会开发，作为其核心项目
  2. IIS
    iis是Internet Information Services的缩写，意为互联网信息服务，由微软公司提供.而IIS是Web（网页）服务组件，其中包括Web服务器、FTP服务器、NNTP服务器和SMTP服务器，分别用于网页浏览、文件传输、新闻服务和邮件发送等方面
  3. APACHE
    Apache是世界使用排名第一的Web服务器软件，是Apache HTTP Server的简称，是Apache软件基金会的一个开放源码的网页服务器

3.实现技术

  1. JSP
    全称JavaServer Pages,是一种动态网页技术标准。JSP以Java语言为脚本语言，为用户的HTTP请求提供服务，并能与服务器上的其它Java程序共同处理复杂的业务需求。
  2. PHP
    即超文本预处理器，是一种通用开源脚本语言。PHP是在服务器端执行的脚本语言，与 C语言类似，是常用的网站编程语言。PHP独特的语法混合了C、Java、Perl以及PHP自创的语法。利于学习，使用广泛，主要适用于Web开发领域。
  3. ASP.NET
    是ASP的升级版，由微软公司开发，
  4. ASP
    ASP即Active Server Pages，是Microsoft公司开发的服务器端脚本环境，可用来创建动态交互式网页并建立强大的web应用程序。

二 浏览器

1.主要功能

  1. 提交请求
    一次完整的浏览器请求流程链接
  2. 作为HTML和脚本的解释器
    解释从服务器端接受的数据，完成事务逻辑。
  3. 以图形化方式显示HTML文档
    将从服务器端收到的数据以图像化的方式展示出来，这个从数据到图形化的转变就是浏览器的解释器的功能的体现。

2.主要产品

  1. Microsoft IE
    微软的IE浏览器
  2. Mozilla Firefox 火狐
  3. Google Chrome
  4. Apple Safari
  5. Opera 欧朋

3.实现技术

  1. HTML
  2. CSS
  3. Javascript
