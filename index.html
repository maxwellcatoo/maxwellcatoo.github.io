HTML入门基础知识？

从知乎上看到的一个关于“若想学HTML，应从何入手？”的回答，虽然答案很劝退，但对于我这种非专业想跨行转计算机的来说，却也是一个补充自己专业基础知识的好机会，故以写此博客为目的查找这些基础知识的概念。
原文链接网页打开需要下拉几下，答案大概排在第5个的样子

WEB基础知识（了解）

一 WEB与Internet

WEB和Internet都被中文翻译成网络的意思，其实，它们各有所指
先说Internet，Internet的意思在这句话中体现的最为明显：“全球性的计算机互联网络，指各个子网所共同连接起来的一个互联的网，重点在于信息共享
而WEB重点指依附于Interne所形成的网页应用等。WEB本意是蜘蛛网和王的意思，表现为三种形式：超文本、超媒体和超文本传输协议
两者之间的关系：
web是Internet上最流行的应用之一，Internet为web提供了网络运行环境
而web的出现极大地推动了Internet的普及和推广

1 Internet

译作因特网，指全球性计算机互联网络，又叫互联网、国际互联网。
网际网络是指在广域网与广域网之间互相连接的网络，互联网的概念也包含在其中，网际网络的概念更抽象一些，指像互联网一样连接起来起来形成的网。

二 Internet提供的服务

主要提供四种服务：
高级浏览WWW服务
电子邮件E-mail服务
远程登录Telent服务
以及文件传输FTP服务

1 高级浏览WEB服务

依附于Internet所形成的的网页应用，主要形式有超文本、超媒体和超文本传输协议

2 电子邮件E-mail服务

字面意思，用邮箱收发邮件的功能，是互联网早期的主要服务之一

3 远程登录Telent

Telent协议是TCP//议族中应用层协议之一，是Interne提供的用来对本地或远端运行的网络设备进行管理、配置、监听和维护，这提高了用户的灵活操作性。模拟管理员进行远程登录好像是以前黑客常用的技术手段，要用到非对称加密技术好像，不太好操作

4 FTP

FTP是与HTTP同层次的在Internet上别广泛使用的协议，HTTP是超文本传输协议，而FTP是文件传输协议，FTP的实现比HTTP的实现要复杂

5 BBS:电子公告板(论坛)

英文Bulletin Board System的缩写，即电子公告板，互联网早期主要应用之一，现在也在被广泛应用着

三 实现技术

1 分组交换原理

就是将用户之间交流的大段的信息拆分成有规定的长度限制的小段，每个小段分为头部和数据段，头部包含有发送方和接收方的地址等信息，而用户传输的数据就在数据段。发送的时候小段在网络中传输不分顺序，在交换机中先缓存起来，等待交换机按照当前的网络状况安排合适的传输线路。等到所有小段都到达目的地后再按顺序整合还愿信息，这就是分组交换原理。

2 TCP/IP 协议簇

先贴一下百度百科中对于这个协议簇的定义：tcp/ip协议簇,可以看到这个协议簇是以TCP/IP协议为主还有其它众多协议一起构建了Internet的基础，包含有SLIP协议、PPP协议、IP协议、ICMP协议、ARP协议、TCP协议、UDP协议、FTP协议、DNS协议、SMTP协议等协议。对这些协议的讲解我将在另外写一篇博客来说明链接。

四 WEB程序

1 什么是WEB

本来想借用上边写的区分WEB和Internet的说明来讲解这个问题，但是看上边写的也是很不具体，那就在这边好好解释一下吧。
WEB主要就是指网页了，在Internet这个将世界各地的局域网搭建成一个更大的、统一的、一个整体的互相连接的一个internet基础上，人们利用HTTP超文本传输协议等一系列协议创建了以文件和超链接将Internet上的各个信息节点连接起来，早期的网页只是由简单的文字和连接，后期虽然图片、音频、视频等表现形式更加直观的文本被展现在网页上，但WEB的基本组成依然还是文本和超链接。

WEB的工作原理

一 基于B/S模式的程序（Web）

1.B/S

B/S就是“Browser/Server”的缩写，B是浏览器的意思，而S是服务器的意思，即浏览器/服务器结构，是对以往的C/S架构的一种改进，也可以说是三层的C/S结构，是一种全新的软件系统构造技术。
第一层：指浏览器，即客户端，只有简单的输入输出功能，本身处理极少的事务逻辑，面向大范围的用户。
第二层：指WEB服务器，帮忙处理第一层客户端与第三层数据库服务器之间的信息调度。
第三层：服务数据库，不仅存储着大量的原始数据，还要处理主要的事务逻辑，是服务器的核心。

2.基于C/S模式的程序

补充一下B/S的前身C/S：
C/S就是“Client/Server”的缩写，C指客户端，S指服务器，所以就是客户端/服务器结构。之前我以为随着B/S的流行，C/S已经落后被淘汰了，其实不是，我们手机中的微信、QQ、王者荣耀等app其实就是C/S结构的。B/S的发展并不是C/S的升级，而是属于从C/S结构中分化出来的针对WEB的高效率的工作的一种分化。

3.二者的区别：

1 硬件环境不同：C/S建立在专门的网络上，有专门的客户端才能访问服务器，而B/S则是建立在广域的网络之上的，任何一款操作系统或浏览器都可以访问服务器。
2 由于上一条两者硬件环境的不同，而导致了本条C/S比B/S结构更安全的不同，因为C/S用户更固定，服务器信息更安全
3 B/S结构维护升级更麻烦一些，动不动就要下载海量的资源包来更新，要完成升级需要客户端和服务器端都升级，而B/S结构升级只需要服务器端完成升级即可。

服务器与浏览器

一 服务器

1.主要功能

给用户提供服务的，根据服务器提供的服务类型不同，分为文件服务器，数据库服务器，应用程序服务器，WEB服务器等

  • 存储WEB信息，提供管理环境
  • 响应浏览器的请求，执行服务器端程序
  • 安全功能

2.主要产品

  1. TOMCAT
    目前比较流行的WEB应用服务器，由APACHE软件基金会开发，作为其核心项目
  2. IIS
    iis是Internet Information Services的缩写，意为互联网信息服务，由微软公司提供.而IIS是Web（网页）服务组件，其中包括Web服务器、FTP服务器、NNTP服务器和SMTP服务器，分别用于网页浏览、文件传输、新闻服务和邮件发送等方面
  3. APACHE
    Apache是世界使用排名第一的Web服务器软件，是Apache HTTP Server的简称，是Apache软件基金会的一个开放源码的网页服务器

3.实现技术

  1. JSP
    全称JavaServer Pages,是一种动态网页技术标准。JSP以Java语言为脚本语言，为用户的HTTP请求提供服务，并能与服务器上的其它Java程序共同处理复杂的业务需求。
  2. PHP
    即超文本预处理器，是一种通用开源脚本语言。PHP是在服务器端执行的脚本语言，与 C语言类似，是常用的网站编程语言。PHP独特的语法混合了C、Java、Perl以及PHP自创的语法。利于学习，使用广泛，主要适用于Web开发领域。
  3. ASP.NET
    是ASP的升级版，由微软公司开发，
  4. ASP
    ASP即Active Server Pages，是Microsoft公司开发的服务器端脚本环境，可用来创建动态交互式网页并建立强大的web应用程序。

二 浏览器

1.主要功能

  1. 提交请求
    一次完整的浏览器请求流程链接
  2. 作为HTML和脚本的解释器
    解释从服务器端接受的数据，完成事务逻辑。
  3. 以图形化方式显示HTML文档
    将从服务器端收到的数据以图像化的方式展示出来，这个从数据到图形化的转变就是浏览器的解释器的功能的体现。

2.主要产品

  1. Microsoft IE
    微软的IE浏览器
  2. Mozilla Firefox 火狐
  3. Google Chrome
  4. Apple Safari

  2. CSS

理解互联网协议

参考文章阮一峰的“互联网协议入门”。
基于“掌握一个知识的最佳督促方式是自己将其复述出来”的原则，将自己对互联网基础协议的理解写出来

互联网协议概述

了解互联网协议可以帮助我们更好的理解前端的各个流程和操作背后所代表的的意义，更是我们深入学习前端的基础。对我来说，更是可以帮助我这个前端小白在阅读其他前端文章时理解相关的术语名词所代表的的含义。

互联网

五层模型

互联网的分层概念首先从设计的角度来看，提高了网络连接操作中的灵活性和包容性。而具体分为五层其实更大程度上是因为这样的分层便于我们理解互联网的作用机理，更大的意义是在于帮助我们理解互联网协议的作用机理，因此，互联网的分层也有分四层和七层的，五层的分层更容易理解，也因此更普遍。

互联网的分层由上到下分别是
应用层（Application Layer）
传输层(Transport Layer)
网络层（Network Layer）
链接层（Link Layer）
实体层（Physical Layer）
对于这五层模型，其英文名称其实更接近于五层模型对各个模型的解释。
由下到上进行理解

实体层（Physical Layer）

physical 的意思是“物质的、身体的、物理的”意思，实体层的主要作用就是用物理手段把互联网中的电脑连接起来，是电脑与电脑进行信息交流的物理层面上的实现，不管是用光缆、电缆、无线电波还是双绞线实现的连接，都为后续层面上的连接提供了可能。从传输完成的概念上来说，实体层只是完成了0/1信号的传输，当然这也是后续一切传输的基础。

链接层（Link Layer）

概念

link 是“关联、一环、桥梁、中继线”的意思，实体层的连接为互联网的概念实现提供了无意义的0/1电信号的传输，链接层则是从0/1电信号入手，制定规则，确定电信号的分组方式。

以太网协议

早期的分组杂乱不堪，这给互联网的连接带来了很大的阻力，后来一个叫做“以太网”的协议占据了主导地位。
以太网规定，一组电信号构成一个数据包，称为以太网数据包，也叫“帧”（Frame),其结构如下：

+------+---------------+
+ Head +      Data     +
+------+---------------+

其中，Head部分称为标头，长度固定，为18字节，包含数据包的一些说明项（这是所有数据包的标头的作用），在这里主要包含发送者的接受者的MAC地址、数据类型等。

MAC地址

每台电脑从生产完成出场之时，只有一个MAC地址，MAC地址长度是48个二进制位，常用12个十六进制数来表示，前六位是厂商编号，后六个是该厂商的网卡流水号，MAC地址是刻在电脑配件中的网卡中的，网卡是电脑连接网络必不可少的设备。对于MAC地址，我们只需要知道这是每台电脑的独一无二的id号就可以了。

广播

广播是是电脑与电脑间传输信息的形式，具体是指发出以太网数据包的电脑会将数据包发送给所有的电脑，接受数据的电恼会对比数据包标头中的接受者MAC地址，若不同，则丢掉，相同的话则接受，这就完成了电脑与电脑之间最基础的信息交流，而且，这样的广播被局限在一定的范围之内，避免了电脑接收到海量的以太网数据包，这个范围就是后边会讲到的局域网。

网络层（Network Layer）

概念

network 是“网状物、网状系统、人际网、网络”的意思，互联网是由无数的子网络共同组成的一个巨型网络，子网络与子网络之间以“路由”的方式发送（路由是个很大的主题，本文不涉及），子网络内则以“广播”的方式进行信息传输，这便是当今互联网的传输方式。子网络也导致了“网络层”的诞生，子网络与子网络之间以“网址”相区分，入网的计算机则将其所在的子网作为自己的网络地址，这样，我们就可以通过先处理计算机的网络地址，在处理计算机的MAC地址来达到在互联网找到一个特定电脑的功能。另外，网址和MAC地址是互相独立的，两者并无联系，可以说，MAC地址是一个人的身份证号，而网址则是一个人的住址

IP协议

网络地址用来区分互联网中的子网，这种区分遵循一定的协议，用来规定网络地址的协议，叫做IP协议，其所定义的地址，称为IP地址

目前，广泛采用的是第四版IP协议：IPv4版本，这个版本规定，网络地址由32个二进制位组成：
10101100.00010000.11111110.00000001
习惯上，我们用分成四段的十进制位来表示：
172.16.254.1
IP地址的范围从0.0.0.0到255.255.255.255

每一台入网的计算机，都有自己的IP地址，但是，计算机的IP地址的网络有的是前24位代表子网络，有的是前32位代表子网络，为了明确判断，引入了“子网掩码”的参数，其与计算机的IP地址位数相对应，属于子网的位数，“子网掩码”全部用1来表示，其余全部用0来表示，这样，通过对两个IP地址与“子网掩码”分别进行AND运算，就可以准确找到每台电脑的IP地址。

IP数据包

根据IP协议发送的数据，叫做IP数据包，IP数据包也分为标头和和数据两部分，IP数据包标头包含版本、长度、IP地址等信息，整个IP数据包都在以太网数据包的数据部分内，结构如下所示

+————————————————————+————————————————+————————————————————————+
+  标头(以太网数据包)  + 标头(IP数据包)  +           数据         +
+————————————————————+————————————————+————————————————————————+

ARP协议

大多数情况下，我们知道对方的IP地址，却不知道对方的MAC地址，难以完成“广播“传输，所以，我们需要一种从IP地址中得到MAC地址的机制。
这里分为两种情况：
1 两台主机不在同一个子网络
2在
第一种情况需要我们将数据包传给子网所在的网关，借用网关将其转换成第二种情况
而ARP协议则是适用于从IP地址得到MAC的机制中的第二种情况的一项协议
ARP协议具体内容：
以广播的形式发出一个包含在以太网数据包中的ARP数据包,其中包含有对方电脑的IP地址，而MAC地址的位置则填上FF:FF:FF:FF:FF:FF,这样子网内所有的电脑都会收到这个包并将自身IP地址与其发送的IP地址对比，相同，则回复自身的MAC地址，不同，则丢弃，由此可完成在同一个子网络之内的电脑之间的交流。

传输层 (Transport Layer)

概念

之前的三层已经完成了互联网内电脑与电脑之间信息传输，但是，下面三层模型并不能分辨出来这些信息属于电脑上的哪个软件，所以，才有了传输层，传输层设定了“端口”的参数，不同的程序拥有不同的端口参数。”传输层”的功能，就是建立”端口到端口”的通信。相比之下，”网络层”的功能是建立”主机到主机”的通信。只要确定主机和端口，我们就能实现程序之间的交流。

UDP协议

遵循UDP协议加入端口信息的数据包称为UDP数据包，其结构如下：

+——————+——————+——————+——————————+
+ head + head + head +   Data   +
+——————+——————+——————+——————————+

三个head分别是以太网数据包的标头、IP数据包的标头和UDP数据包的标头

TCP协议

UDP协议的优点是比较简单，容易实现，但是缺点是可靠性较差，一旦数据包发出，无法知道对方是否收到。TCP协议应运而生，TCP协议每发一个数据包都会确认，假如在传输过程中，有哪怕一个数据包丢失，发出方也能知道数据传输失败，从而重新进行发送。（但是TCP也因此变得十分复杂，实现困难，且消耗较多的资源）

应用层（Application Layer）

应用层的作用，在于规定应用程序的数据格式，其数据就放在TCP数据包的数据部分。
